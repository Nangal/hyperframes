✱ consistently · eval-driven

How a new skill goes live.

Every new skill or feature comes online through an eval-driven process, run end-to-end by our agents — four steps:

01 Understand The need from our builders — including ourselves.
02 Reverse Take the most representative video and reverse-decompose it into skills + input.
03 Generalize Build a benchmark; solve across more inputs until it generalizes.
04 Eval Pairwise / pointwise comparison or judge — until the skill is verified.

✱ eval-driven · example: circle highlight

Worked example: the circle highlight.

Circle / ring highlight is one of the most common asset animations. The hard part is location — cloud models (Claude Opus included) are unreliable at exact pixel coordinates, so the ring drifts off the subject. Here's how we ground it.

The fix — a numbered-grid locate loop · RSVP grid-index, on Opus

The model picks grid strips (columns & rows), never raw coordinates — then crops and grids again to lock on.

Generalizes — Opus across a 7-case test set · native box (red) drifts → grid box (green) locks on · 21.3% → 3.7% avg
